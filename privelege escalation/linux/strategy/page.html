Enumeration

1. Check your user (id, whoami)
2. Run Linux Smart Enumeartion
3. Run LinEnum & other scripts as well
4. Manual command (https://blog.g0tmi1k.com/2011/08/basic-linux-privilege-escalation/)


Strategy

Spend some time and read over the result of your enumeration.

if Linux Smart Enumeration level 0 or 1 finds something intersting, mae a note of it.
Avoid rabbit holes by creating a checklist of things you need for the privilege escalation method to work (requirement for privesc).


Have a quick look around for files in user's home directory and other commont location: /var/backup /var/logs

If user has a history file, read it, it may have important information like commands or even passwords.

Try less steps first:
* Sudo
* Cron Jobs
* SUID files


Have a good look at root processes, enumerate their versions and search for exploits.


Check for internal prots that you might be able to forward to your attacking machine.

Re-read full enumeration dumps and highlight anything that seems odd. process or file name you aren't familiar with, an "unusual" filesystem configured (on Linux, anything that isn't ext, swap, or tmpfs), or even a username.

Thinking about kernel exploits.
